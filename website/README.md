# Native UI Kit docs site

Docusaurus 3 site published to GitHub Pages on every push to `main` and on
every `v*.*.*` tag. Source for the public docs at
`https://clevertap.github.io/clevertap-native-ui-kit/`.

## Local dev

```bash
cd website
npm install
npm start            # http://localhost:3000/clevertap-native-ui-kit/
```

## Populating the API reference locally

The API reference (Dokka HTML for Android, DocC HTML for iOS) is **not
committed** — it's regenerated by CI on every deploy. To preview it
locally:

```bash
# From the repo root — Android (Dokka)
cd android && ./gradlew :sdk:dokkaHtml
mkdir -p ../website/static/api/android/1.0.0
cp -R sdk/build/dokka/html/. ../website/static/api/android/1.0.0/

# iOS (DocC) — requires Xcode (macOS only)
cd ../ios
xcodebuild docbuild -scheme CleverTapNativeDisplay \
    -destination 'generic/platform=iOS Simulator' \
    -derivedDataPath docc-build
ARCHIVE=$(find docc-build -name '*.doccarchive' | head -1)
xcrun docc process-archive transform-for-static-hosting "$ARCHIVE" \
    --output-path ../website/static/api/ios/1.0.0 \
    --hosting-base-path /clevertap-native-ui-kit/api/ios/1.0.0
```

Without these, `Android API` and `iOS API` navbar links show empty pages
locally — the rest of the site renders fine.

## Production build

```bash
npm run build        # outputs to website/build/
```

The production build is what CI ships. It runs `docusaurus build` and
expects the API reference dirs to be populated (CI handles this via
artifact downloads).

## Versioning

Versioned snapshots are committed under `versioned_docs/` and
`versioned_sidebars/`. Cut a new snapshot once before a release:

```bash
npx docusaurus docs:version 1.1.0
```

This freezes `docs/` into `versioned_docs/version-1.1.0/`. CI does this
automatically when a tag matching `v*.*.*` is pushed; the docs:version
snapshot only commits if it doesn't already exist for that tag.

## Search

Local search ships with `@easyops-cn/docusaurus-search-local`. Algolia
DocSearch is a future option — apply at https://docsearch.algolia.com/
once the docs are publicly indexed.
